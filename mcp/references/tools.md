# 工具参数与行为参考

## 服务连接

### HTTP API
- API Base URL：`/api/v1/skills/miliastra-knowledge/tools`
- 例子：`https://ugc.nfasystem.top/api/v1/skills/miliastra-knowledge/tools`
- 调用方式：HTTP `POST` + JSON 请求体

### MCP Server
- 连接地址：`https://ugc.nfasystem.top/mcp`
- 协议：streamable-http
- 支持 4 个工具：`get_node_info` / `list_documents` / `get_document` / `rag_search`
- 配置到 `~/.claude/settings.json`：
  ```json
  {
    "mcpServers": {
      "miliastra-knowledge": {
        "type": "url",
        "url": "https://ugc.nfasystem.top/mcp"
      }
    }
  }
  ```

**Linux调用示例**：

Windows powershell请先对应更新成相应的语法
```bash
curl -X POST https://ugc.nfasystem.top/api/v1/skills/miliastra-knowledge/tools/get_node_info \
  -H "Content-Type: application/json" \
  -d '{
    "names": ["碰撞触发器", "嘲讽目标"]
```

```javascript
const response = await fetch('/api/v1/skills/miliastra-knowledge/tools/get_node_info', {
```

## `get_node_info`

### 参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `names` | `list[str]` | 是 | 节点名称列表，支持模糊匹配，可批量传入 |

### 行为

- **子串匹配**：query 是 target 的子串即命中，如 `"运动器"` 命中 `"投射运动器"`
- **子序列匹配**：query 的每个字符按顺序出现在 target 中即命中，如 `"觉醒"` 可命中 `"角色技能觉醒"`
- 两种方式任一命中即返回，**不区分大小写**
- 单次调用可传多个名称，结果按传入顺序逐项返回

### 返回结构

```json
[
  {
    "query": "定点位移",
    "matches": [
      {
        "title": "定点位移",
        "main_title": "一、角色技能",
        "source_doc_title": "执行节点",
        "local_path": "official/guide/mh277t9fl4tm_执行节点.md",
        "output_file": "derived/node/执行节点.md",
        "content": "..."
      }
    ]
  }
]
```

## `list_documents`

### 参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `keywords` | `list[str]` | 否 | 过滤关键词列表，支持模糊匹配；空列表时返回全部文档 |

### 行为

- `keywords` 为空列表时返回全部文档（guide + tutorial + FAQ），共 270+ 条
- 传入多个关键词时逐个过滤，**按关键词分组**返回各自结果，结构为 JSON array
- 对 title 和文件名均做子串/子序列匹配
- 返回结果只含 `title` 和 `file` 路径，不含正文（正文需用 `get_document`）

### 返回结构

**单关键词**（`keywords=["运动器"]`）：
```json
[
  {
    "keyword": "运动器",
    "total": 3,
    "documents": [
      {"title": "基础运动器",  "file": "official/guide/xxx_基础运动器.md"},
      {"title": "投射运动器", "file": "official/guide/mhaqt9rgqv4u_投射运动器.md"},
      {"title": "跟随运动器", "file": "official/guide/xxx_跟随运动器.md"}
    ]
  }
]
```

**多关键词**（`keywords=["运动器", "造物"]`）：
```json
[
  {"keyword": "运动器", "total": 3, "documents": [...]},
  {"keyword": "造物",   "total": 6, "documents": [...]}
]
```

**无关键词**（`keywords=[]` 或不传）：
```json
{
  "total": 274,
  "documents": [{"title": "读前须知", "file": "..."}, ...]
}
```

## `get_document`

### 参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `titles` | `list[str]` | 是 | 文档标题或关键词列表，支持模糊匹配，可批量传入 |

### 行为

- 文件系统只扫描一次，所有 titles 共享同一候选列表，批量时无额外开销
- 每个 title 独立进行子串/子序列匹配，结果按传入顺序逐项返回
- 每项结果附带 `status` 字段：
  - `"ok"` — 命中 1–5 篇，返回全文 + related_nodes
  - `"too_many"` — 命中 >5 篇，仅返回标题列表，需用更精确关键词重查（宽泛词如 `"节点"` 会触发）
  - `"not_found"` — 未命中，返回 available_titles_sample 供参考
- `related_nodes` 由同一 title 关键词触发节点模糊匹配，可能为空列表

### 返回结构

```json
[
  {
    "query": "碰撞触发器",
    "status": "ok",
    "documents": [
      {
        "title": "碰撞触发器",
        "file": "official/guide/mh8w69rzuc3i_碰撞触发器.md",
        "content": "---\ntitle: 碰撞触发器\n...",
        "related_nodes": [{"title": "进入碰撞触发区", "content": "..."}]
      }
    ]
  },
  {
    "query": "不存在的标题",
    "status": "not_found",
    "message": "未找到匹配「不存在的标题」的文档",
    "available_titles_sample": ["读前须知", "碰撞触发器", "..."],
    "related_nodes": []
  },
  {
    "query": "节点",
    "status": "too_many",
    "message": "匹配到 8 篇文档，请用更精确的关键词。",
    "matches": [{"title": "执行节点", "file": "..."}, "..."],
    "related_nodes": [...]
  }
]
```

## `rag_search`

### 参数

| 字段 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| `queries` | `list[str]` | 是 | — | 自然语言问题或描述列表，越具体越精准，可批量传入 |
| `top_k` | `int` | 否 | 5 | 每个查询最多返回 top_k 条结果 |

### 行为

- DB 连接和环境变量只初始化一次，批量调用时共享，逐 query 执行嵌入 + 向量检索
- `similarity` 低于 0.3 的结果被静默过滤，`total_results` 可能小于 `top_k`
- 结果为空的 query 返回空 `results` 数组，不报错，不影响其他 query 结果
- 对含错别字或描述模糊的 query 仍可返回合理结果，但精确描述效果更好

### 返回结构

```json
[
  {
    "query": "角色受击后播放特效",
    "total_results": 4,
    "results": [
      {
        "title": "特效播放",
        "h1_title": "执行节点",
        "file_name": "mh4ppo02m1o8_特效播放.md",
        "similarity": 0.812,
        "text_snippet": "播放限时特效节点可在指定位置播放一次性特效..."
      }
    ]
  },
  {
    "query": "第二个查询",
    "total_results": 2,
    "results": [...]
  }
]
```
